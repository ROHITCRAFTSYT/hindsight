## LinkedIn

**Most AI assistants have a hangover problem.**

You give one all the context it needs, it nails the answer — and the next session it remembers nothing. LLMs are stateless, and bolt-on RAG gives you recall without understanding.

For the WeMakeDevs × @Cognee hackathon I built **Hindsight**: a second-brain copilot on Cognee's memory layer that makes an AI's memory something you can actually *watch*.

🟢 Remember — drop in notes, files, or URLs
🔵 Recall — ask anything in natural language
🟣 Improve — 👍/👎 feedback re-weights memory
🔴 Forget — prune stale facts with one click

…all on top of a live, force-directed knowledge graph that grows and shrinks as the memory changes. Cognee stores a hybrid graph + vector memory, so the agent reasons over *relationships*, not just similar-looking text.

It runs on Cognee Cloud or fully self-hosted (SQLite + LanceDB) via a single env var, and ships a zero-key demo mode so anyone can try it instantly. Open source under MIT.

The takeaway from building it: once memory is a first-class API — remember / recall / improve / forget — a huge amount of agent plumbing just disappears.
